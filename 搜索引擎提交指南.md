# 搜索引擎提交操作指南

本站已配置 **sitemap.xml**、**robots.txt** 以及各页的 **meta 描述与 canonical**，并为主站新闻生成了**静态 HTML 页面**，便于被 Google、Yahoo、百度等收录。请按下面步骤在各大搜索引擎的「站长工具」中提交站点和 sitemap。

---

## 一、Google（谷歌）

1. **打开 Google Search Console**  
   - 网址：<https://search.google.com/search-console>

2. **添加/选择资源**  
   - 若尚未添加站点：点击「添加资源」→ 选择「网域」或「网址前缀」。  
   - **网域**：输入 `newlife-dev-group.com`（需在 DNS 里已配置 TXT 验证）。  
   - **网址前缀**：输入 `https://newlife-dev-group.com`，若你之前用 `googleb419375949f853e3.html` 验证过，选该资源即可。

3. **提交 Sitemap**  
   - 左侧菜单进入 **「站点地图」**（Sitemaps）。  
   - 在「添加新的站点地图」处输入：`sitemap.xml`  
   - 点击「提交」。  
   - 状态变为「成功」后，Google 会按 sitemap 抓取你的页面。

4. **（可选）请求编入索引**  
   - 左侧「网址检查」→ 输入首页或重要新闻页 URL（如 `https://newlife-dev-group.com/` 或某条新闻的 `https://newlife-dev-group.com/news/n_xxxx.html`）。  
   - 点击「请求编入索引」，可加快该页被收录的速度。

---

## 二、Yahoo（雅虎） / Bing（必应）

日本等地的 **Yahoo 搜索由 Bing 提供技术支持**，在 **Bing Webmaster Tools** 中验证并提交后，通常也会对 Yahoo 搜索生效。

1. **打开 Bing Webmaster Tools**  
   - 网址：<https://www.bing.com/webmasters>

2. **添加站点**  
   - 点击「Add a site」或「添加站点」。  
   - 输入：`https://newlife-dev-group.com`

3. **验证所有权**  
   - 任选一种方式（推荐「导入从 Google Search Console」若已验证过 GSC）：  
     - **导入 Google Search Console**：用同一 Google 账号登录并授权，可一键导入已验证的站点。  
     - **XML 文件**：下载 Bing 提供的 XML 文件，上传到站点根目录（与 `googleb419375949f853e3.html` 同级），然后点击验证。  
     - **Meta 标签**：将提供的 meta 标签放入首页 `<head>` 中后验证。

4. **提交 Sitemap**  
   - 验证通过后，在「Sitemaps」里提交：`https://newlife-dev-group.com/sitemap.xml`  
   - 或只填：`sitemap.xml`

5. **（可选）URL 提交**  
   - 在「URL Submission」中可提交重要 URL，促进抓取。

---

## 三、百度（Baidu）

1. **打开百度搜索资源平台**  
   - 网址：<https://ziyuan.baidu.com/>  
   - 使用百度账号登录（无则先注册）。

2. **添加网站**  
   - 进入「用户中心」→「站点管理」→「添加网站」。  
   - 输入：`https://newlife-dev-group.com`（或 `http://newlife-dev-group.com`，按你实际使用为准）。

3. **验证网站**  
   - 任选其一：  
     - **文件验证**：下载百度提供的 `baidu_verify_xxxx.html`，上传到站点根目录（与 `sitemap.xml` 同级），然后点击「完成验证」。  
     - **HTML 标签验证**：把给出的 meta 标签插入首页 `<head>` 中，保存后点击验证。  
     - **CNAME 验证**：按提示在域名 DNS 添加 CNAME 记录。

4. **提交 Sitemap**  
   - 验证通过后，在「普通收录」或「链接提交」里：  
     - 选择「sitemap」方式，提交：`https://newlife-dev-group.com/sitemap.xml`  
   - 也可在「手动提交」中粘贴重要页面 URL（如首页、新闻列表、几条新闻详情页），每日有配额。

5. **（建议）主动推送（可选）**  
   - 若使用「API 提交」或「主动推送」，需在百度获取接口地址与 token，将 sitemap 中的 URL 批量推送，收录会更快（适合有技术能力时使用）。

## 五、本站已完成的配置摘要

| 项目 | 说明 |
|------|------|
| **sitemap.xml** | 根目录下，包含首页、各事业页、新闻列表、每条可访问的新闻静态页。 |
| **robots.txt** | 根目录下，`Allow: /`，并声明 `Sitemap: https://newlife-dev-group.com/sitemap.xml`。 |
| **静态新闻页** | 位于 `news/` 目录，如 `news/n_20260212_1.html`，正文在首屏 HTML 内，便于收录。 |
| **meta 与 canonical** | 各 HTML 页已添加 `meta name="description"` 与 `link rel="canonical"`。 |

**新增新闻时（本地一键更新后统一推送）：**

1. 在 **`news-data.json`** 中按现有格式添加一条记录（id、type、date、tag、title、content 等）。  
2. 在 **`yangnan1987.github.io`** 目录下执行一条命令，完成「静态新闻页 + sitemap」的更新：  
   ```bash
   npm run build
   ```  
   （或直接执行：`node build-news-static.js`）  
   脚本会为所有「お知らせ / 活動レポート / 研究レポート」生成/更新 `news/<id>.html`，并**自动重写 sitemap.xml**，无需再手动改 sitemap。  
3. 本地检查无误后，按平时流程推送到 Git 即可部署：  
   ```bash
   git add .
   git commit -m "更新新闻与 sitemap"
   git push
   ```  
4. （可选）部署后若希望新页面更快被收录，可在 Google / Bing / 百度 站长工具中对新 URL 做一次「请求编入索引」或「URL 提交」。

---

## 六、收录需要时间

- 提交 sitemap 和 URL 后，**收录通常需要数天到数周**，新站或更新较少时可能更久。  
- 可在各站长工具中查看「覆盖率」「已编入索引的 URL」等状态。  
- 若长时间未收录，可检查：站点是否可被外网访问、robots.txt 是否未误拦、sitemap 中的 URL 是否可正常打开。

完成以上步骤后，你的 NEWS 被 Google、Yahoo、百度等搜索到的概率会明显提高。
