# 新闻 / 研究レポート 更新规范

## 日期规则（必读）

- **新增新闻或研究レポート时，日期必须使用「当天」日期。**
- AI 助手无法获取实时日期，因此：
  - 若**用户已说明**「今天是 YYYY.M.D」，则**必须**使用该日期（例如：今天是 2026.2.16 → 使用 `2026.2.16`）。
  - 若用户未说明今天日期，**应先询问用户当天日期**，再写入 `news-data.json`。
- 请勿依赖系统或对话中的“今日日期”提示，以**用户明确告知的日期**为准。

## news-data.json 格式

- 新条目放在数组**最前**（按时间倒序）。
- `date` 格式：`YYYY.M.D`（例如 `2026.2.16`）。
- 研究レポート：`tag` 为 `研究レポート`，`type` 为 `event`。

## 静态页与发布（必读）

- 修改或新增 `type` 为 `internal` / `event` 的条目后，在本目录执行 **`node build-news-static.js`**，生成 `news/*.html` 并更新 `sitemap.xml`。
- **完成上述修改与构建后**：自行检查 JSON / 静态页 / 资源路径等无明显错误，**直接在 `yangnan1987.github.io` 仓库内 `git commit` 并 `git push`**，无需等用户再发「推送」指令（仅限本站新闻相关变更；其他仓库或敏感操作仍按用户单独说明为准）。
